프로젝트

IT 개발자 지망생을 위한 온라인 스터디 서비스

역할

  • 프론트엔드
  • 피그마에 맞추어 다양한 컴포넌트(달력,슬라이드배너,타이머 등등) 직접 개발
  • Axios 활용해 데이터 가져오기

배운점

  • 라이브러리를 사용하지 않고 컴포넌트들을 개발하는 것이 더 좋을때가 있다는 점을 깨달음
  • 다른 사람들이 컴포넌트를 쓸 수 있게끔 하려면, 말뿐만 아니라 문서가 필요함을 느낌
  • 기획단계(피그마)에서의 모습과 실제 개발화면에서의 모습이 생각보다 많이 달라 디자인이 자주 바뀔 수 있다는 점을 깨달음

아쉬운 점

  • 컴포넌트에 변수를 너무 많이 넣어서, 만들어 놓고 쓰지 못하는 일이 발생하기도 함
  • 백엔드 쪽의 일을 많이 도와주지 못함

1면에 개인 성향에 맞추어 추천뉴스를 보여주는 서비스

역할

  • 백엔드(SpringBoot)
  • Spring Security를 이용해 로그인 구현
  • Kakao 소셜 로그인 구현

배운점

  • 로그인의 구현에 대해서 깊게 알게 됨
  • 다양한 폴더 구성법에 대해서 장점과 단점을 알게됨(계층형 폴더,기능형 폴더)

아쉬운 점

  • 네이버의 소셜 로그인 허용기준이 높다는 것을 로직을 다 만들고 나서 알아 1주일이란시간을 낭비함
  • 시간이 없어 Redis를 사용해보지 못함
  • 파이썬이나 배포와 관련된 일에 대해선 거의 손을 대지 못함

싸이월드 UI 기반의 개발자 커뮤니티 서비스

역할

  • 백엔드(SpringBoot)
  • CRUD와 관련된 모든 것(게시글,댓글,친구)

배운 점

  • Spring과 JPA 이용해 온전히 CRUD 개발
  • 성능을 크게 상관하지 않으면 다 개발할 수 있다는 자신감이 생김

  • 성능 개선을 할 수 있는 방향이 많아 보이는데 시도해 보지 못함
  • elastic Search 같은 기술을 사용해 보지 못함
